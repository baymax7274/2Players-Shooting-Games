# v1.2.0 动态环境系统 — 设计文档

> 日期：2026-05-10 | 分支：动态地图元素 | 基于 v1.1.1

## 目标

为现有双人对战射击游戏增加地图动态元素，提升每局战斗的不可预测性和策略深度。核心原则：**地图从静态背景变为影响战局的活体**。

## 架构

### 新增/修改文件

```
src/systems/pickup_manager.py   [新] 拾取物管理器
src/systems/environment.py      [新] 环境事件系统
src/entities/pickup.py          [新] 拾取物实体
src/entities/map.py             [改] tile破坏、移动墙、动态瓦片
data/maps/*.json                [改] 5张地图配置扩展
src/scenes/battle.py            [改] 集成新系统
config.py                       [改] 拾取物/环境常量
```

### 数据流

```
BattleScene.update(dt)
  ├─ PickupManager.update(dt, players, game_map)   → 刷新/碰撞/效果
  ├─ Environment.update(dt, game_map, players)      → 平台/灾害/专属事件
  ├─ GameMap.update(dt)                             → 破坏物/移动墙/tile变更
  └─ (原有射击/玩家/回合逻辑不变)
```

三个新模块各自独立，通过 BattleScene 协调，不修改原有系统内部逻辑。

## 功能设计

### 1. 拾取物系统 (PickupManager + Pickup)

地图预设刷新点，战斗中定时刷新拾取物，玩家走过即拾取。

| 类型 | 效果 | 刷新间隔 | 同时上限 | 视觉 |
|------|------|----------|---------|------|
| 血包 | +25 HP | 20s | 3 | 红色十字 |
| 伤害翻倍 | 1.5x伤害，持续8s | 45s | 1 | 紫色光球 |
| 加速 | 1.5x移速，持续6s | 40s | 1 | 绿色闪电 |

- 每张地图配置 8-12 个刷新点，随机选取空闲点位
- 强力道具（伤害/加速）全局最多同时 1 个
- 拾取触发粒子特效 + 音效

**常量（config.py）：**
```
PICKUP_RESPAWN_HEALTH = 20.0
PICKUP_RESPAWN_POWER = 45.0
PICKUP_RESPAWN_SPEED = 40.0
PICKUP_MAX_HEALTH = 3
PICKUP_MAX_POWER = 1
PICKUP_MAX_SPEED = 1
```

**PickupManager 接口：**
- `update(dt, players, game_map)` — 刷新计时、检测碰撞、应用效果
- `render(screen, offset)` — 渲染活跃拾取物
- `reset()` — 新回合清空
- `get_spawn_points(map_name)` — 从地图配置读取刷新点

### 2. 可破坏环境 (GameMap 扩展)

#### 瓦片类型完善

| tile | 含义 | 特性 |
|------|------|------|
| 0 | 空地 | 不变 |
| 1 | 永久墙壁 | 不变 |
| 2 | 可破坏墙 | 爆炸类武器/手雷可摧毁 |
| 3 | 半高掩体 | 子弹可越过，阻挡移动 |
| 4 | 移动不可破坏墙 | 沿路径往复，不可摧毁 |

#### 破坏规则

- 火箭弹、手雷爆炸 → 爆炸半径内 tile=2 全部摧毁
- 霰弹枪近距离（<100px）→ 单发摧毁 tile=2
- 摧毁时：碎块粒子（8-12 棕色方形碎片）+ 屏幕微震 + 音效
- tile 变为 0，wall_rect 从碰撞列表移除

#### 半高掩体（tile=3）

- 渲染高度 16px（正常一半）
- 阻挡移动
- 子弹碰撞时判断弹道高度，高于掩体顶部则穿过

#### 移动墙（tile=4）

- 沿预设路径往复移动
- 到达端点暂停 0.5s 后反向
- 速度 60-100 px/s
- 碰到玩家：墙推人
- 子弹命中：与普通墙一致（阻挡/反弹）

**地图 JSON 新增配置项：**
```json
{
  "pickup_spawns": [{"x": 15, "y": 20}, ...],
  "moving_walls": [
    {
      "tile_x": 20, "tile_y": 15, "tile_w": 2, "tile_h": 3,
      "path": [{"x": 20, "y": 15}, {"x": 20, "y": 25}],
      "speed": 80,
      "pause": 0.5
    }
  ]
}
```

**GameMap 新增方法：**
- `update(dt)` — 驱动移动墙
- `destroy_tile(tx, ty)` → 返回粒子数据
- `get_tiles_in_radius(cx, cy, radius)` → tile 坐标列表
- `rebuild_wall_rects()` — 破坏/移动后重建碰撞列表

### 3. 地图专属机制 (Environment 系统)

每张地图有独特的动态事件，通过 Environment 类统一调度。

| 地图 | 机制 | 参数 |
|------|------|------|
| 训练靶场 | 移动掩体 ×2 | 场地中央两块横向往复墙 |
| 城市废墟 | 随机塌方 | 每25-35s 一块边缘 tile=2 自动坍塌 |
| 地下堡垒 | 毒气扩散 | 回合60s后四角产生毒雾（半径3格），每15s扩大1格 |
| 太空站 | 重力反转 | 每15s 重力方向上下翻转（利用已有 gravity_mod） |
| 密林迷宫 | 荆棘陷阱 | 预设荆棘块，每20s随机激活3处持续5s，触碰扣15血 |

**配置驱动：** 每张地图在 JSON 中声明 `environment` 字段启用/配置专属机制，Environment 根据当前地图名加载对应逻辑。

**Environment 接口：**
- `setup(map_name, config)` — 根据地图加载机制
- `update(dt, game_map, players)` — 驱动所有活跃机制
- `get_active_hazards()` — 返回当前灾害区域（供渲染）
- `reset()` — 新回合重置

### 4. 渲染变更

- 移动墙：深蓝灰色 (60,70,90)，表面画方向箭头
- 可破坏墙：爆炸/命中时闪白 0.1s 再生成碎片
- 毒雾区域：半透明绿色覆盖，边缘虚化
- 荆棘陷阱：红色尖刺贴图闪烁
- 重力反转提示：屏幕边缘短暂箭头闪烁

## 边界条件

- 拾取物不刷新在墙体内（刷新点必须在 tile=0 位置）
- 移动墙路径不能与其他墙体重叠
- 毒雾/荆棘不对死亡玩家造成伤害
- 重力反转时子弹物理同步切换重力方向
- 新回合开始：拾取物清空、破坏墙恢复、移动墙归位
- 退出到主菜单时所有状态被销毁（不需要持久化）

## 影响范围

- **不修改**：武器系统、经济系统、HUD、音效系统、粒子系统（仅调用）
- **轻量修改**：battle.py 增加 3 个 update/render 调用；player.py 增加 power-up 状态字段
- **新增**：3 个新模块 + 5 张地图配置扩展
